# Chess Academy Management System
An end-to-end role-based platform for managing a chess academy.

This project is a full-stack web application built to simplify daily academy operations such as student registration, batch scheduling, attendance tracking, fee handling, tournament management, and role-based access for different users.

## Overview

The system is designed for four main user roles:

1. Admin
2. Clerk
3. Coach
4. Parent

Each user sees a different dashboard and is allowed to perform only the actions that belong to their role.

## Key Features

1. Secure authentication using JWT
2. Role-based dashboards and protected routes
3. Student registration and student management
4. Automatic unique student ID generation
5. NIC-based student uniqueness validation
6. Batch creation, assignment, and enrollment
7. Attendance tracking for students and coaches
8. Payment and finance handling
9. Tournament registration and approval flow
10. Notifications and alerts
11. Admin announcements CRUD
12. Search, filtering, and reporting

## How To Run The Project

### 1. Prerequisites

Make sure the following are installed:

- Python 3.12 or compatible 3.x version
- Node.js 18+ or newer
- MySQL 8.0
- Git

### 2. Clone The Repository

```bash
git clone https://github.com/your-username/chess-academy-management-system.git
cd chess-academy-management-system
```

### 3. Backend Setup

Go to the backend folder:

```powershell
cd backend
```

Create and activate a virtual environment:

```powershell
python -m venv venv
.\\venv\\Scripts\\activate
```

Install backend dependencies:

```powershell
pip install -r requirements.txt
```

### 4. Configure Environment Variables

Update `backend/.env` with your local values.

Example:

```env
DJANGO_SECRET_KEY=your_secret_key_here
USE_SQLITE=false
DB_NAME=chess_academy_db
DB_USER=chess_admin
DB_PASSWORD=your_mysql_password
DB_HOST=localhost
DB_PORT=3306
EMAIL_HOST_USER=your_email@example.com
EMAIL_HOST_PASSWORD=your_email_app_password
```

Important:
- Set `USE_SQLITE=false` if you want to run with MySQL.
- If you only want a quick local run without MySQL, set `USE_SQLITE=true`.

### 5. Run Backend Migrations

```powershell
python manage.py migrate
```

### 6. Start Backend Server

```powershell
python manage.py runserver
```

Backend will run at:

```text
http://localhost:8000/
```

### 7. Frontend Setup

Open a new terminal and go to the frontend folder:

```powershell
cd frontend
```

Install frontend packages:

```powershell
npm install
```

Start the frontend:

```powershell
npm start
```

Frontend will run at:

```text
http://localhost:3000/
```

## Default API Base URL

```text

```

- `frontend/.env`

## Sample Login Flow

1. User enters email or username and password.
2. Frontend sends the request to `/api/auth/login/`.
3. Backend checks the MySQL database.
4. If credentials are valid, JWT access and refresh tokens are returned.
5. Frontend stores the tokens and redirects the user to the correct dashboard.

## Important Backend Behaviors

1. Student IDs are automatically generated by the backend.
2. NIC is validated as a unique 12-digit number.
3. Admin can approve or reject pending student registrations.
4. Admin can manage announcements through a small CRUD module.
5. Role-based permissions protect sensitive operations.

## Troubleshooting

### 1. `ModuleNotFoundError: rest_framework`

Install backend dependencies again:

```powershell
pip install -r requirements.txt
```

### 2. Database connection error

Check `backend/.env` and confirm:

- MySQL service is running
- `DB_USER` is correct
- `DB_PASSWORD` is correct
- `DB_NAME` exists

### 3. Login shows invalid email or password

Make sure the backend is connected to the correct database and that you restarted the server after changing `.env`.

### 4. Frontend cannot reach backend

Make sure both servers are running:

- backend on `8000`
- frontend on `3000`

### 5. Migration errors

Run:

```powershell
python manage.py makemigrations
python manage.py migrate
```

## Notes For New Clones

If someone clones this repository for the first time:

2. Install backend requirements.
3. Run Django migrations.
4. Install frontend dependencies.
5. Start both servers.
